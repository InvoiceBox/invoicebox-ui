# 04 — Скорость и отзывчивость

Для библиотеки «производительность» = (а) размер кода, доезжающего до бандла
потребителя, (б) tree-shakability, (в) отсутствие тяжёлого в критическом пути,
(г) визуальная стабильность компонентов (CLS, мерцание скелетонов).

---

### P-1. CJS-only сборка без tree-shaking
- **Где:** `babel.config.json` (`"modules": "commonjs"`), `package.json`
  (`main: ./dist/index.js`, нет `module`/`exports`/`sideEffects`).
- **Серьёзность:** **high**.
- **Описание:** пакет публикуется только в CommonJS, единым barrel-ом. Потребитель,
  импортирующий один `TextInput`, получает в бандл весь граф (`react-calendar`,
  `qrcode.react`, `react-modal-sheet`, маски и т.д.), т.к. CJS не шейкается.
- **Решение:** добавить ESM-сборку и `"exports"` map с подпутями
  (`@invoicebox/ui/TextInput`), проставить `"sideEffects": false` (с осторожностью к
  styled-components). Перейти на бандлер библиотек (`tsup`/`rollup`/`vite lib`).
- **Эффект:** кратное снижение размера у потребителей, реальный tree-shaking.

### P-2. Мёртвая зависимость `motion` (~framer-motion) в проде
- **Где:** `package.json` dependencies `motion ^12.38.0`; использований в `src` = **0**.
- **Серьёзность:** high (по соотношению вес/польза).
- **Описание:** одна из самых тяжёлых анимационных библиотек объявлена прод-зависимостью
  и не используется. Тянется в граф установки потребителя.
- **Решение:** удалить из `dependencies` (после подтверждения, что не нужна).
- **Эффект:** минус крупная зависимость из дерева, чище установка.

### P-3. Нет измерения размера бандла per-component
- **Где:** процесс сборки.
- **Серьёзность:** medium.
- **Описание:** размер `dist` 2.4 MB (транспиляция без минификации), но реального
  per-export бюджета нет — регрессии веса не отлавливаются.
- **Решение:** `size-limit` с бюджетами на ключевые экспорты в CI; `esbuild --metafile`
  для анализа графа.
- **Эффект:** контроль регрессий размера на каждом PR.

### P-4. Тяжёлые компоненты в общем графе (Calendar/QRCode/масками)
- **Где:** `react-calendar` (7 файлов), `qrcode.react`, `@mona-health/react-input-mask`,
  `react-custom-scrollbars-2`.
- **Серьёзность:** medium.
- **Описание:** редко используемые тяжёлые компоненты (QRCode, Calendar) попадают в граф
  даже когда не нужны (следствие P-1).
- **Решение:** после P-1 — изоляция через подпути-exports + опциональный `React.lazy`
  для демонстраций; вынести в peer/optional где уместно.
- **Эффект:** критический путь форм не тянет календарь/QR.

### P-5. CLS / мерцание — Skeleton и Drawer
- **Где:** `common/Skeleton`, `common/Drawer/index.tsx` (задержка размонтирования 300мс),
  загрузочные состояния форм.
- **Серьёзность:** medium.
- **Описание:** скелетоны без фиксированных размеров вызывают сдвиг макета (CLS) при
  подмене на контент; множественные независимые скелетоны → «скелетон-шторм» и мерцание.
  Нужен единый загрузчик с задержкой появления (anti-flicker) и резервированием места.
- **Решение:** Skeleton с обязательными `width/height`; глобальная политика «единый
  загрузчик с delay 150–200мс и min-display» против мерцания; зарезервированные размеры
  у всех плейсхолдеров.
- **Эффект:** CLS→~0, нет мельтешения при быстрых загрузках.

### P-6. Анимации без `prefers-reduced-motion`
- **Где:** `prefers-reduced-motion` = 0 совпадений; анимации в Drawer/Toast/спиннерах.
- **Серьёзность:** medium (пересечение с a11y, см. `05`).
- **Описание:** анимации всегда включены — нагрузка на слабые устройства + проблема
  доступности.
- **Решение:** общий хук/CSS-обёртка, отключающая анимации при `prefers-reduced-motion: reduce`.
- **Эффект:** меньше работы рендера на слабых девайсах, выполнение WCAG 2.3.3.

---

**Приоритет блока:** P-1 + P-2 (high) — волна 1 (измеримый эффект); P-3/P-5 — волна 2;
P-4/P-6 — волна 3.
