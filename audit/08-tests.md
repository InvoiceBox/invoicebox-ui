# 08 — Тесты и CI

Текущее покрытие близко к нулю: **1 тест-файл, 4 теста** (`websiteValidate.test.ts`),
на ~38 компонентов и весь модуль `validate`. CI нет.

---

### T-1. Нет CI вообще
- **Где:** `.github/workflows` отсутствует.
- **Серьёзность:** **high**.
- **Описание:** ни тесты, ни линт, ни `audit`, ни сборка не проверяются автоматически.
  Любой PR может слить регрессию; релиз идёт без гейта (см. PL-5).
- **Решение:** GitHub Actions: на PR — `install → lint → typecheck → test --coverage →
  build → npm audit (порог)`; на тег — релиз. Матрица node 18/20.
- **Эффект:** автоматический контроль качества каждого изменения.

### T-2. Критический путь без тестов
- **Где:** `validate/utils/vatNumberValidateHelpers.ts` (ИНН/ОГРН/БИК/р-с СНГ),
  `validate/index.ts` (контрольные суммы settlementAccount/correspondentAccount/ОГРН),
  `form/PhoneInput/phoneInputLogic.ts`, `CurrencyInput`, `DateInput`/`DateTimeInput`.
- **Серьёзность:** **high**.
- **Описание:** бизнес-критичная логика (банковские реквизиты, контрольные числа,
  телефоны, деньги, даты) почти не покрыта. Ошибка здесь = неверная валидация платёжных
  данных у всех потребителей.
- **Решение:** unit-тесты на все валидаторы (включая edge: пустые/мусорные/граничные
  длины, NaN), на phone/currency/date-логику. Это самый высокий ROI.
- **Эффект:** защита денежно-критичной логики, ловля регрессий.

### T-3. Нет компонентных/integration-тестов
- **Где:** компоненты `form/*`, `common/Drawer`, `Select`, `Autocomplete`.
- **Серьёзность:** medium.
- **Описание:** нет проверки рендера, взаимодействий, доступности.
- **Решение:** React Testing Library + `jest-dom` + `user-event`; `jest-axe` для a11y-
  регрессий (синергия с `05`); Storybook play-functions (`@storybook/test` уже в devDeps,
  но не используется) для interaction-тестов.
- **Эффект:** покрытие поведения и доступности ключевых компонентов.

### T-4. Jest гоняет тесты дважды (из `dist/`)
- **Где:** `jest` без конфига → подхватывает `dist/validate/utils/websiteValidate.test.js`.
- **Серьёзность:** low.
- **Описание:** дублирующий прогон скомпилированной копии; зашумляет вывод, маскирует
  расхождения src/dist.
- **Решение:** `jest.config` с `testPathIgnorePatterns: ['/dist/','/node_modules/']`,
  `coverageThreshold`, окружение `jsdom` для компонентов.
- **Эффект:** чистый, детерминированный прогон + пороги покрытия.

### T-5. Нет e2e критического пути
- **Где:** —
- **Серьёзность:** low (для библиотеки).
- **Описание:** полноценный e2e в UI-kit избыточен; роль e2e выполняют Storybook
  interaction-тесты и визуальная регрессия.
- **Решение:** опционально — визуальная регрессия (Chromatic/Playwright-screenshots) по
  stories; в CI на PR.
- **Эффект:** ловля визуальных регрессий без тяжёлого e2e-стека.

---

**Целевые пороги (предложение):** старт — coverage по `validate/` ≥80%; общий ≥50% к
концу волны 2, ≥70% к волне 3.

**Приоритет блока:** T-1 + T-2 (high) — волна 1; T-3/T-4 — волна 2; T-5 — волна 3.
