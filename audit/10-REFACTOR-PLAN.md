# План рефакторинга

**Ни одно изменение кода не происходит из этого документа без явного согласования по каждой волне.** Этот план не затрагивает бизнес-логику или публичный API/поведение какого-либо компонента — всё ниже это внутренние вещи (пайплайн сборки, тесты, ARIA-атрибуты, конфиг линтера, версии зависимостей). Если какой-то пункт ниже на практике потребует изменения API или поведения, это отдельно проговаривается и требует отдельного согласования до начала работ — это не проезжает автоматически вместе с планом.

Волны упорядочены: риски/измеримость → скорость → удобство/доступность → фундамент, согласно ТЗ. Каждая волна должна попадать отдельным PR (или небольшим набором PR), после неё нужно перезамерить показатели по [01-BASELINE-METRICS.md](01-BASELINE-METRICS.md) и получить go/no-go перед началом следующей волны.

## Что не трогается

Ни одна волна не затрагивает публичные API компонентов, форму props или визуальное/поведенческое поведение, наблюдаемое потребителем — единственное исключение, разбиение `AppCardLink` для tree-shaking (Волна 1), требует решения, является ли исправление внутренним (перестроен объект-lookup, тот же публичный API) или новой формой публичного API (экспорты по варианту) — **этот конкретный выбор требует согласования до начала Волны 1**, поскольку второй вариант — breaking change для существующих потребителей.
