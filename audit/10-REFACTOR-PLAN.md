# 10 — План рефакторинга (волнами)

Принцип очерёдности: **сначала риски и измеримость** — сперва ставим «сетку
безопасности» (CI, тесты, измерения), затем закрываем высокий риск, затем качество и
доступность. Бизнес-логику и пользовательские пути **не меняем без согласования**
(отдельное продуктовое решение).

> Каждая волна = отдельная ветка/PR. Документацию синхронизируем в том же PR, что и код.
> После каждой волны — повторный замер метрик из `01-BASELINE-METRICS.md`.

---

## Волна 0 — «Сетка безопасности» (риск + измеримость, без правок бизнес-логики)
Цель: начать измерять и не дать регрессий. Минимум изменений поведения.

1. **CI** (T-1, PL-5): GitHub Actions — lint + typecheck + test + build + `npm audit`.
2. **Jest-конфиг** (T-4): `testPathIgnorePatterns: /dist/`, `coverage`, `coverageThreshold`,
   jsdom.
3. **Измерения** (P-3): `size-limit` бюджеты на ключевые экспорты; baseline зафиксирован.
4. **Lint-гейты дёшево** (AR-2, AR-6): `max-lines` (warn), `eslint-plugin-jsx-a11y`,
   `@storybook/addon-a11y` — пока как warn, чтобы увидеть масштаб A-блока.
5. **Удалить мёртвое** (P-2, AR-6): убрать `motion`; заменить shim-полифилы
   `object.entries/fromentries` нативными.

**Эффект/метрика:** появляется CI-гейт; coverage и size измеряются; −1..3 зависимости.
**Риск:** минимальный (инфраструктура).

## Волна 1 — Высокий риск: безопасность + платформа + критич. тесты
1. **S-1**: `isSafeUrl` + `rel="noopener noreferrer"` в `AppCardLink`.
2. **S-2**: правый якорь и JSDoc для `Validate.website`, ReDoS-чек.
3. **PL-2 + S-3**: миграция Storybook 7 → 8/9 (Vite), выпил `react-scripts`/CRA-пресета →
   падение `npm audit` на десятки CVE.
4. **PL-3**: `engines.node >=18`, `.nvmrc`, CI-матрица.
5. **T-2**: unit-тесты на весь `validate/` + phone/currency/date-логику (ROI максимум).

**Эффект/метрика:** `npm audit` critical→0, coverage `validate/` ≥80%, закрыт XSS-вектор.
**Риск:** средний (Storybook-миграция); бизнес-логика не трогается.

## Волна 2 — Производительность сборки + типы + наблюдаемость
1. **P-1 + PL-4**: переход на `tsup`/`rollup`: ESM+CJS, `exports`-map с подпутями,
   `sideEffects`, минификация; убрать `dist` из git, собирать в CI.
2. **PL-1**: апгрейд TS 5 / ESLint 9 / typescript-eslint 8.
3. **AR-1**: убрать `any`/unsafe assertions, включить `no-explicit-any`/`no-unsafe-*`.
4. **O-1/O-2**: единый dev-логгер + опциональный `onError`.
5. **T-3**: RTL + `jest-axe` + Storybook play-тесты на ключевые компоненты.

**Эффект/метрика:** tree-shakable пакет (замер size до/после), типобезопасный API,
coverage ≥50%.
**Риск:** средний (смена сборщика — проверить совместимость экспортов у потребителей).

## Волна 3 — Доступность (по компонентам) + качество + резилентность
> Самый объёмный блок; вести покомпонентно, каждый компонент — отдельный небольшой PR
> со stories + a11y-тестом.
1. **A-1/A-2/A-3** (critical/high): клавиатура+ARIA для Select/Dropdown/Autocomplete/
   Toggle; label↔control+ошибки; фокус-трап в Drawer. Рассмотреть Radix/React Aria
   как базу примитивов.
2. **A-4/A-5/A-6**: контраст палитры, alt у SVG, `prefers-reduced-motion`, `rem`/reflow.
3. **P-5/P-6**: Skeleton с резервированием размеров (CLS), anti-flicker загрузчик.
4. **R-1/R-2/R-4/R-5**: ErrorBoundary, guard-парсинг, безопасный `useLoadingSubmit`,
   fallback палитры.
5. **AR-3/AR-5, O-3/O-4**: ререндеры, стайлгайд API, CHANGELOG, экспорт `version`.

**Эффект/метрика:** `jest-axe`/addon-a11y без нарушений на покрытых компонентах;
CLS≈0; coverage ≥70%.
**Риск:** изменения поведения компонентов — **каждый a11y-PR, меняющий разметку/фокус,
согласуется** (может затронуть верстку потребителей).

## Что требует явного согласования перед стартом
1. **Storybook-миграция 7→8/9** (волна 1) — затрагивает dev-процесс команды.
2. **Смена сборщика и `exports`-map** (волна 2) — меняет способ импорта у потребителей
   (обратная совместимость barrel-импорта сохраняется, подпути добавляются).
3. **A11y-изменения разметки/ролей** (волна 3) — могут влиять на стили/селекторы
   потребителей; продуктовое решение по приоритету компонентов.
4. **Удаление `motion`** — подтвердить, что не планируется к использованию.
