# 05 — Доступность (WCAG 2.1 AA)

Самый слабый блок. По всему `src`: `aria-*` = **0**, `role=` = **0**,
обработчиков клавиатуры (`onKeyDown/Up/Press`) = **2**, `prefers-reduced-motion` = **0**.
Для библиотеки **форм-инпутов** это критично — недоступность тиражируется во все
приложения-потребители.

---

### A-1. Кастомные интерактивные компоненты не работают с клавиатуры
- **Где:** `form/Select`, `common/Dropdown`, `form/Autocomplete`, `form/Toggle`,
  `form/DateInput`/`DateTimeInput`, `form/CodeInput`, `common/Drawer`.
- **Серьёзность:** **critical**.
- **Описание:** dropdown/select/autocomplete построены на `div`-ах без клавиатурной
  навигации (стрелки/Enter/Esc/Home/End), без `tabindex`, без управления фокусом.
  Пользователь клавиатуры/скринридера не может ими пользоваться. Нарушение
  WCAG 2.1.1 (Keyboard), 4.1.2 (Name/Role/Value).
- **Решение:** реализовать паттерны WAI-ARIA Authoring Practices:
  - Select/Combobox → `role="combobox"` + `aria-expanded`/`aria-controls`/
    `aria-activedescendant`, список `role="listbox"`, опции `role="option"`,
    навигация стрелками, Enter/Esc.
  - Toggle → нативный `<input type="checkbox/switch">` или `role="switch"` +
    `aria-checked`, активация Space/Enter.
  - Рассмотреть Radix UI / React Aria как базу примитивов (доступность «из коробки»).
- **Эффект:** полноценная работа с клавиатуры и скринридерами во всех потребителях.

### A-2. Поля без программной связи label↔control и сообщений об ошибке
- **Где:** `form/InputLabel`, `form/TextInput`, `form/PureInput`, все поля.
- **Серьёзность:** **high**.
- **Описание:** нет гарантии `htmlFor`/`id` связи, ошибки валидации не объявляются
  (`aria-invalid`, `aria-describedby` на текст ошибки). Скринридер не сообщает причину
  ошибки. WCAG 1.3.1, 3.3.1, 3.3.2, 4.1.3.
- **Решение:** `InputLabel` генерирует/прокидывает `id`, поле получает `aria-invalid`,
  ошибка — `role="alert"` + связь через `aria-describedby`.
- **Эффект:** доступная валидация форм, корректное озвучивание ошибок.

### A-3. Модалки/Drawer без фокус-трапа и возврата фокуса
- **Где:** `common/Drawer/index.tsx` (на `react-modal-sheet`), `hooks/useModal`.
- **Серьёзность:** **high**.
- **Описание:** нет гарантии фокус-трапа внутри открытого Drawer, возврата фокуса на
  триггер при закрытии, `role="dialog"`/`aria-modal`/`aria-labelledby`, закрытия по Esc.
  WCAG 2.1.2 (No keyboard trap наоборот — нужен управляемый трап), 2.4.3 (Focus Order).
- **Решение:** focus-trap (focus-trap-react или примитив диалога Radix), возврат фокуса,
  `aria-modal="true"` + `aria-labelledby` на заголовок, Esc-закрытие, блокировка фона.
- **Эффект:** доступные модальные сценарии, корректный порядок фокуса.

### A-4. Контраст и размеры тач-таргетов не верифицированы
- **Где:** `palette/constants.ts`, `Typography`, кнопки/чипы/иконки.
- **Серьёзность:** medium.
- **Описание:** нет проверки контраста текста/фона (≥4.5:1 для обычного текста) и
  минимального размера интерактивных целей (≈44×44px). WCAG 1.4.3, 2.5.5.
- **Решение:** прогнать палитру через контраст-чекер, задокументировать допустимые пары;
  гарантировать min-таргеты в стилях кнопок/иконок-кнопок.
- **Эффект:** соответствие AA по контрасту и удобство на тач-устройствах.

### A-5. Иконки/SVG без текстовых альтернатив
- **Где:** `common/InvoiceboxLogo`, `Flag`, `WarningIcon`, `Arrow`,
  `AppCardLink/components/*` (GooglePlay/Apk/RuStore/AppGallery), `QRCode`.
- **Серьёзность:** medium.
- **Описание:** значимые SVG без `role="img"`/`aria-label` или `aria-hidden` для
  декоративных. Ссылки-картинки магазинов без доступного имени. WCAG 1.1.1.
- **Решение:** значимым — `aria-label`/`<title>`, декоративным — `aria-hidden="true"`;
  `AppCardLink` — доступное имя ссылки.
- **Эффект:** скринридер корректно описывает графику и ссылки.

### A-6. Нет учёта `prefers-reduced-motion` и масштабирования
- **Где:** анимации (Drawer/Toast/спиннеры), вёрстка в `px`.
- **Серьёзность:** medium.
- **Описание:** анимации не отключаются по `reduce`; жёсткие `px` мешают
  масштабированию до 200% (WCAG 1.4.4, 1.4.10 Reflow, 2.3.3 Animation from Interactions).
- **Решение:** media-query `prefers-reduced-motion`; перевод критичных размеров на
  `rem`; проверка reflow при zoom 200%.
- **Эффект:** комфорт для чувствительных к движению и слабовидящих пользователей.

---

**Приоритет блока:** A-1 (critical), A-2/A-3 (high) — волны 1–2 (по компонентам);
A-4/A-5/A-6 — волна 3. Рекомендация: подключить `eslint-plugin-jsx-a11y` и
`@storybook/addon-a11y` уже в волне 1 как измеримый гейт.
