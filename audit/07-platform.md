# 07 — Платформа (версии, сборка, CVE, кэш)

---

### PL-1. Устаревший инструментарий (TS / ESLint / typescript-eslint)
- **Где:** `package.json` — TypeScript `4.9.5`, ESLint `8.x`, `@typescript-eslint/* 5.x`.
- **Серьёзность:** medium.
- **Описание:** TS 4.9 (текущие — 5.x), ESLint 8 (EOL), typescript-eslint 5 (старая
  мажорка) — нет современных правил (`no-unsafe-*`), типов и багфиксов.
- **Решение:** апгрейд до TS 5.x, ESLint 9 (flat config) или хотя бы 8 latest,
  typescript-eslint 8. Прогон автофиксов, актуализация конфигов.
- **Эффект:** современные проверки типов/линта, основа для AR-1.

### PL-2. Storybook 7 тянет уязвимую `react-scripts`/CRA-цепочку (источник CVE)
- **Где:** devDeps `@storybook/preset-create-react-app`, `@storybook/* 7.6`,
  транзитивно `react-scripts` → `@svgr/*`, `workbox-*`, `rollup-plugin-terser`, `svgo`.
- **Серьёзность:** **high**.
- **Описание:** именно CRA-пресет тянет основную массу high/critical из `npm audit`.
  CRA задепрекейчен.
- **Решение:** мигрировать на Storybook 8/9 на Vite-билдере, выкинуть
  `preset-create-react-app`/`react-scripts`. Это снимет десятки CVE разом.
- **Эффект:** резкое падение `npm audit`, быстрее dev-сборка витрины.

### PL-3. Заявленный `engines.node >=16` недостоверен
- **Где:** `package.json` `"engines": { "node": ">=16.0.0" }`; локально node 16.20.2.
- **Серьёзность:** medium.
- **Описание:** Storybook 7+ и современный tooling требуют node 18+; node 16 — EOL
  (с 2023). Заявленная совместимость вводит в заблуждение и ломает воспроизводимость.
- **Решение:** поднять до `>=18` (лучше `>=20`); добавить `.nvmrc`/`engine-strict`;
  зафиксировать версию в CI-матрице.
- **Эффект:** воспроизводимая сборка, поддерживаемый рантайм.

### PL-4. Сборка не воспроизводима и без бандлинга
- **Где:** `release.sh` (Babel CLI транспиляция, без минификации/бандла),
  `dist/` закоммичен в репозиторий.
- **Серьёзность:** medium.
- **Описание:** артефакт собирается ad-hoc, коммитится в git (раздувает историю),
  нет ESM/минификации (см. P-1). Babel-транспиляция «как есть».
- **Решение:** перейти на `tsup`/`rollup` (ESM+CJS+минификация+`exports`), убрать `dist`
  из git (`.npmignore`/`files` уже ограничивает публикацию — но коммит лишний),
  собирать в CI.
- **Эффект:** воспроизводимый, минифицированный, tree-shakable артефакт.

### PL-5. Релиз вручную с локальной машины, без гейта
- **Где:** `release.sh` — `git checkout main && merge develop && npm publish` локально.
- **Серьёзность:** **high** (операционный риск).
- **Описание:** публикация в npm и пуш в `main` идут с машины разработчика без прогона
  тестов/линта/audit, без 2FA-гейта, без provenance. Риск выката битого/непроверенного
  пакета и человеческой ошибки в git.
- **Решение:** CI-релиз (GitHub Actions) с тегов: lint+test+audit+build → `npm publish`
  с `--provenance`, OIDC-токеном вместо локального; `main` под branch protection.
- **Эффект:** надёжный, проверяемый, аудируемый релиз.

### PL-6. Кэширование статики витрины
- **Где:** Storybook на GitHub Pages (`docs/`, `CNAME ui.invoicebox.ru`).
- **Серьёзность:** low.
- **Описание:** GitHub Pages даёт ограниченный контроль заголовков кэша; хешированные
  ассеты Storybook кэшируются, но `index.html` стоит держать `no-cache`.
- **Решение:** при переходе на собственный хостинг — immutable-кэш для хешированных
  ассетов, `no-cache` для html. На Pages — оставить как есть (вне контроля).
- **Эффект:** корректная инвалидция витрины (низкий приоритет).

**Приоритет блока:** PL-2 + PL-5 (high) — волна 1; PL-1/PL-3/PL-4 — волна 2; PL-6 — фоном.
